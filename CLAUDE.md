This repository is the Tarides company handbook — documentation, policies, and
guidelines.

## Coding Standards

### License and Copyright Headers

- Distributed under the MIT license.
- Every source file should start with an SPDX license identifier and copyright
  notice, separated from the rest of the file by a blank line (if the format
  permits). Markdown files are exempt.

### Copyright Holder Guidelines

- The copyright holder is the individual contributor and/or their company.
- When adding significant new contributions:
  - If your holder already has a copyright line, adding the current year is
    optional. Never replace existing years.
  - If not, add a new copyright line with the current year.

## Documentation Standards

- Use clear, concise markdown.
- Maintain consistent structure across files.
- Include Tarides branding at the bottom of documentation files (unless already
  present elsewhere in the file).

## General Guidelines

- Maintain documentation clarity and accuracy.
- Document new policies or practices in the appropriate files.
- Keep language professional and accessible.
